# 📝 Guía para Colaborar

---

## 🙌 ¡Gracias por tu interés!

Este manual es un proyecto abierto a la comunidad. Tu colaboración ayuda a mejorar el contenido, corregir errores y ampliar el alcance del trabajo. Antes de contribuir, por favor lee esta guía.

---

## 🔍 ¿Cómo puedo colaborar?

### 1. Reportar errores
Si encuentras un error tipográfico, de contenido, legal o ético, abre un **issue** en GitHub describiendo el problema.

**Qué incluir:**
- El archivo donde está el error.
- Una descripción clara del error.
- Una sugerencia de corrección (si la tienes).

### 2. Sugerir mejoras
Si crees que falta contenido, que algún módulo se puede ampliar o que hay una herramienta que debería incluirse, abre un **issue** con tu sugerencia.

**Qué incluir:**
- El módulo o archivo al que afecta.
- Una descripción de la mejora.
- Fuentes o referencias que respalden la sugerencia.

### 3. Aportar contenido
Si quieres escribir o ampliar un archivo, haz un **fork** del repositorio, trabaja en tu copia y envía un **pull request**.

**Normas para el contenido:**
- Usa un lenguaje claro, profesional y respetuoso.
- Cita las fuentes cuando sea posible.
- Respeta el estilo y formato del manual.
- Asegúrate de que el contenido es ético y legalmente correcto.

### 4. Mejorar la documentación
Si ves que algún archivo no está bien explicado, que falta un ejemplo o que el formato es mejorable, abre un issue o pull request.

---

## 📋 Normas de estilo

### Formato de archivos
- Todos los archivos usan **Markdown (.md)**.
- Títulos: usar `#` para el título principal, `##` para secciones, `###` para subsecciones.
- Listas: usar `-` o `*` para viñetas, `1.` para listas numeradas.
- Tablas: usar el formato estándar de Markdown.

### Estilo de escritura
- Usar **lenguaje inclusivo** y no sexista.
- Evitar el sensacionalismo.
- Ser objetivo y basado en hechos.
- Citar fuentes cuando sea posible.
